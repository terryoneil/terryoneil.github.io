*[repo]: repository
*[GUI]: Graphical User Interface
*[commit]: A saved snapshot of your changes, with a message describing the change.
*[clone]: The act of downloading a copy of a repo to your computer so you can edit it without affecting the original. 
*[fork]: Your personal copy of somebody else's Github repository.
*[syntax]: The set of formal rules that defines the structure and format of a coding or markup language. You can think of syntax as the "grammar" of coding languages.
*[stage]: To select a changed file to include in your next commit.
*[local commit]: A commit that isn't pushed to the main branch. It lives only on your local machine.
*[pulling]: Downloading the latest changes from the GitHub repo to your local device. If a teammate has pushed changes since your last sync, pulling will bring those changes to your local copy.
